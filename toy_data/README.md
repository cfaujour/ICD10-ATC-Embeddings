# Toy pipeline (reproducible example)

This folder contains a fully reproducible toy example of the embedding pipeline described in the paper.
It illustrates how joint semantic embeddings for **ICD-10 diagnoses** and **ATC medications** are learned
from longitudinal code sequences using co-occurrence statistics and matrix factorization.

The toy pipeline is intended for **methodological transparency and pedagogical purposes**.
It does not require access to any real claims data.

The pipeline is implemented as four standalone R scripts that are meant to be run sequentially.
Running the scripts creates a `toy_data/data/` subfolder containing all intermediate and final outputs.

---

## Overview of the pipeline

The toy pipeline follows the same conceptual steps as the real-data analysis:

1. Generation of synthetic EHR-like sequences of diagnosis and medication codes, mimicking the structure
   of the French claims data (ESND) used in the paper
2. Computation of code co-occurrences within a fixed temporal window
3. Learning of low-dimensional code embeddings using **PPMI + truncated SVD**
4. Visualization of the resulting embedding space using a UMAP projection

All scripts use repo-root relative paths and write their outputs to a flat `toy_data/data/` directory.

---

## Scripts and outputs

### 1) `01_dataset_toy_gen.R` — Toy data generation

Generates synthetic longitudinal sequences of mixed ICD-10 and ATC codes.
A `code_topics` file is also produced to encode the latent structure used to generate the toy data
(e.g., diagnosis–treatment associations), which is useful for interpretation.

**Main outputs:**
- `sequences.csv.gz` — toy EHR-like code sequences
- `code_topics.csv` — latent topic structure used for data generation

---

### 2) `02_co_occ_calc.R` — Co-occurrence counting and vocabulary

Computes code co-occurrence counts within a sliding window along the toy sequences
and constructs a vocabulary assigning a unique integer index to each code.

**Main outputs:**
- `vocab.csv` — mapping between codes and integer indices
- `cooc_matrix.rds` — sparse co-occurrence matrix (R `dgCMatrix`)

---

### 3) `03_embed.R` — Embedding learning (PPMI + SVD)

Transforms the co-occurrence matrix into a PPMI matrix and applies truncated SVD
to obtain low-dimensional, static embeddings for all codes.

**Main outputs:**
- `embeddings.csv.gz` — learned embeddings (indexed)

Embeddings are indexed; the mapping back to ICD-10 and ATC codes is provided by `vocab.csv`.

---

### 4) `04_embed_viz.R` — Embedding visualization

Produces a low-dimensional visualization of the learned embedding space
to illustrate that related diagnoses and medications cluster together
in the toy setting.

**Main outputs:**
- `umap_2d_toy.png` — 2D UMAP visualization of the toy embeddings
